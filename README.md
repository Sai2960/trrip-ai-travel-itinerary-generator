## 📌 Overview

**Trrip.ai** is a full-stack AI-powered web application built as an assignment for **Orbitra Technologies** (Junior Full Stack Developer — MERN + AI role). Users can upload their travel booking documents (flight tickets, hotel confirmations, travel passes), and the app automatically extracts booking information using **Google Gemini AI** to generate a rich, structured day-by-day travel itinerary — beautifully presented and fully shareable.

## ⚙️ Getting Started

### Prerequisites

- Node.js v18+
- A [Google Gemini API Key](https://aistudio.google.com/app/apikey)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/Sai2960/trrip-ai-travel-itinerary-generator.git
cd trrip-ai-travel-itinerary-generator

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env
```

### Environment Variables

Create a `.env` file in the root with the following:

```env
GEMINI_API_KEY=your_gemini_api_key_here
APP_URL=http://localhost:3000
JWT_SECRET=your_jwt_secret_here
```

### Running the App

```bash
# Run both frontend and backend in development
npm run dev
```

The app will be available at `http://localhost:5173` (frontend) and `http://localhost:3000` (backend API).

## 🧠 How It Works

```
User uploads document (PDF / Image)
        ↓
Gemini AI extracts travel data
(departure, arrival, dates, hotel, flight no., etc.)
        ↓
User reviews extracted info + sets trip dates
        ↓
Gemini AI generates full day-by-day itinerary
(morning / afternoon / evening for each day)
        ↓
Itinerary saved → viewable in History
        ↓
User shares via unique public link
```

## 🔌 API Endpoints

| Method | Endpoint | Description | Auth |
|---|---|---|---|
| `POST` | `/api/auth/register` | Register a new user | ❌ |
| `POST` | `/api/auth/login` | Login & receive JWT | ❌ |
| `POST` | `/api/upload` | Upload travel document | ✅ |
| `POST` | `/api/itinerary/generate` | Generate AI itinerary | ✅ |
| `GET` | `/api/itinerary/history` | Get all user itineraries | ✅ |
| `GET` | `/api/itinerary/:id` | Get single itinerary | ✅ |
| `GET` | `/api/shared/:id` | View shared itinerary | ❌ |
